逛小红书的时候看到了这么一篇帖子，呀～一下子就有点感慨了。
![](https://pic.lamper.top/wp/2026/20260313225845_290_20.webp)

毕竟现在在公司也是用 AI 用得很多了。
虽然大的功能一般不会让 AI 写，但具体的业务逻辑需求还是自己理清楚之后再写代码。

不过对于一些工具性质的代码，现在基本也会交给 AI 生成，然后自己再去验证和测试。

---

一下子就想起了去年负责公司综合支付系统开发的时候。

接手的时候是真的战战兢兢。毕竟支付这种东西，只要和钱有关，就必须严谨再严谨。

这个项目基本就没怎么用 AI。
毕竟 AI 也不是万能的。

从那天开始，我浏览器的标签页就一个接一个地开。

特别是遇到服务商模式的微信 V3 支付对接的时候。
最离谱的是：付款码支付居然没有在 SDK 里面实现。

当时是真的把：

- 博客园
- CSDN
- GitHub Issue
- 微信支付社区

几乎翻了个遍，都没有找到完整的实现。

最后才终于找到了一个官方调用地址。

然后我就开始一点点去看微信支付 SDK 的源码。研究它是怎么做验签的，看它内部是怎么调用接口的

最后才自己写了一个调用方法。

再想想2021年大一那会。
每天晚上基本都在刷黑马的 Java 视频学习。
真就是对着视频敲一遍代码这样学过来的,遇到问题了也是各种网上查找文档留言希望有好心人哈哈哈，现在还翻到了StackOverflow和CSDN的提的问题！

到了2022 年 ChatGPT 出现并火起来的时候，当时其实我是有点不屑一顾的。 想： 一个 AI 能有多 NB？ 结果火了半年之后，好奇去试了一下。 然后才开始慢慢改观。
